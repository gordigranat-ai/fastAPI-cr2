# FastAPI Контрольная работа №2

## Описание проекта

Данный проект представляет собой серверное веб-приложение, разработанное с использованием современного фреймворка FastAPI. Приложение реализует полноценный REST API с системой аутентификации, управлением пользователями, каталогом продуктов и обработкой HTTP заголовков. 

Проект создан в рамках контрольной работы №2 и демонстрирует навыки разработки серверных приложений, включая валидацию данных, работу с cookie, криптографическую подпись и управление сессиями.

---

## Функциональность приложения

Приложение предоставляет следующий функционал:

### Управление пользователями
- Создание новых пользователей с валидацией данных
- Проверка корректности email адресов
- Валидация возраста (только положительные числа)

### Каталог продуктов
- Получение информации о продукте по идентификатору
- Поиск продуктов по ключевым словам
- Фильтрация результатов по категориям
- Ограничение количества выдаваемых результатов

### Система аутентификации
- Вход в систему с проверкой учетных данных
- Установка защищенных HTTP-only cookie
- Хранение сессий пользователей
- Защищенные маршруты, доступные только авторизованным пользователям

### Безопасность
- Криптографическая подпись cookie с использованием библиотеки itsdangerous
- Защита от подделки данных сессии
- Автоматическое завершение сессий по истечении времени
- Динамическое продление сессий при активности пользователя

### Работа с HTTP заголовками
- Извлечение и обработка заголовков User-Agent и Accept-Language
- Валидация формата заголовков
- Установка пользовательских заголовков в ответах сервера

## Технические характеристики

### Архитектура
Приложение построено на модульной архитектуре с разделением ответственности:

- **Маршрутизация** - отдельные роутеры для разных групп эндпоинтов
- **Модели данных** - Pydantic модели для валидации и сериализации
- **Бизнес-логика** - обработка запросов в соответствующих модулях
- **Зависимости** - переиспользуемые компоненты для проверки аутентификации

### Хранение данных
В учебных целях используется in-memory хранение:
- Словарь с тестовыми учетными записями пользователей
- Список продуктов для каталога
- Словарь для хранения активных сессий

### Безопасность
- HTTP-only cookie для защиты от XSS-атак
- Криптографическая подпись для предотвращения подделки
- Автоматическое истечение сессий через 5 минут
- Защита от ручного продления сессий

## Технологический стек

| Компонент | Технология | Назначение |
|-----------|------------|------------|
| Веб-фреймворк | FastAPI 0.104.1 | Создание REST API |
| Сервер | Uvicorn 0.24.0 | Запуск ASGI-приложения |
| Валидация | Pydantic 2.11.0 | Модели данных и проверка |
| Криптография | Itsdangerous 2.2.0 | Подпись cookie |
| Email валидация | Email-validator 2.2.0 | Проверка email адресов |
| Form-data | Python-multipart 0.0.20 | Обработка форм |
